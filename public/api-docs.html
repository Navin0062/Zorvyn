🔐 Authentication

All authenticated endpoints require a JWT token in the Authorization header:

Authorization: Bearer <your-jwt-token>
POST /api/auth/register

Register a new user account

Request Body

Field Type Required Description
email string Yes Valid email address (must be unique)
password string Yes Minimum 6 characters
name string Yes User's full name
role string No viewer, analyst, or admin (default: viewer)
Example Request
POST /api/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123",
  "name": "John Doe",
  "role": "viewer"
}
Example Response (201 Created)
{
  "message": "User registered successfully",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "John Doe",
    "role": "viewer",
    "is_active": true,
    "created_at": "2026-04-05T10:00:00Z"
  }
}
POST /api/auth/login

Authenticate user and receive JWT token

Request Body

Field Type Required Description
email string Yes Registered email address
password string Yes User's password
Example Request
POST /api/auth/login
Content-Type: application/json

{
  "email": "admin@zorvyn.com",
  "password": "admin123"
}
Example Response (200 OK)
{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "uuid",
    "email": "admin@zorvyn.com",
    "name": "Admin User",
    "role": "admin",
    "is_active": true
  }
}

👥 User Management

Manage system users. Most operations require admin privileges.

GET /api/users 🔒 Auth Required

List all users with pagination

Query Parameters

Parameter Type Required Description
page integer No Page number (default: 1)
limit integer No Items per page (default: 10)
search string No Search by name or email
Example Response (200 OK)
{
  "message": "Users retrieved successfully",
  "data": [
    {
      "id": "uuid",
      "email": "admin@zorvyn.com",
      "name": "Admin User",
      "role": "admin",
      "is_active": true,
      "created_at": "2026-04-05T10:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 3,
    "totalPages": 1
  }
}
POST /api/users 👑 Admin Only

Create a new user

Request Body

Field Type Required Description
email string Yes Valid email address
password string Yes Minimum 6 characters
name string Yes User's full name
role string No viewer, analyst, or admin
PUT /api/users/:id 👑 Admin Only

Update user details

Request Body (all optional)

Field Type Description
name string Updated name
role string viewer, analyst, or admin
is_active boolean Active status
DELETE /api/users/:id 👑 Admin Only

Delete a user

PATCH /api/users/:id/toggle-status 👑 Admin Only

Toggle user active/inactive status

💰 Financial Records

Manage financial transactions and entries.

GET /api/records 🔒 Analyst+

List financial records with filtering

Query Parameters

Parameter Type Required Description
page integer No Page number (default: 1)
limit integer No Items per page (default: 10)
type string No income or expense
category string No Filter by category
startDate string No Start date (YYYY-MM-DD)
endDate string No End date (YYYY-MM-DD)
search string No Search description or category
POST /api/records 🔒 Analyst+

Create a new financial record

Request Body

Field Type Required Description
amount number Yes Positive number
type string Yes income or expense
category string Yes Category name
date string Yes Date in YYYY-MM-DD format
description string No Notes or description
Example Request
POST /api/records
Authorization: Bearer <token>
Content-Type: application/json

{
  "amount": 1500.50,
  "type": "income",
  "category": "Salary",
  "date": "2026-04-05",
  "description": "Monthly salary payment"
}
Example Response (201 Created)
{
  "message": "Financial record created successfully",
  "record": {
    "id": "uuid",
    "amount": 1500.50,
    "type": "income",
    "category": "Salary",
    "date": "2026-04-05",
    "description": "Monthly salary payment",
    "user_id": "uuid",
    "created_at": "2026-04-05T10:00:00Z"
  }
}
GET /api/records/:id 🔒 Analyst+

Get a specific financial record

PUT /api/records/:id 👑 Admin Only

Update a financial record

Request Body (all optional)

Field Type Description
amount number Updated amount
type string income or expense
category string Updated category
date string Updated date
description string Updated description
DELETE /api/records/:id 👑 Admin Only

Delete a financial record (soft delete)

🔒 Role-Based Access Control

The system implements three role levels with different permissions:

Action Viewer Analyst Admin
View dashboard summary ✓ ✓ ✓
View own records ✓ ✓ ✓
View all records ✗ ✓ ✓
Create records ✗ ✓ ✓
Update records ✗ ✗ ✓

Manage users ✗ ✗ ✓

⚠️ Error Handling

The API uses standard HTTP status codes and returns consistent error responses.

HTTP Status Codes

Code Meaning Example
200 Success Successful GET, PUT request
201 Created New resource created (POST)
400 Bad Request Validation error, invalid input
401 Unauthorized Missing or invalid token
403 Forbidden Insufficient permissions
404 Not Found Resource doesn't exist
409 Conflict Duplicate entry (e.g., email)
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Unexpected server error

Error Response Format

{
  "error": "Validation failed",
  "details": [
    {
      "type": "field",
      "value": "invalid-email",
      "msg": "Invalid value",
      "path": "email",
      "location": "body"
    }
  ]
}
